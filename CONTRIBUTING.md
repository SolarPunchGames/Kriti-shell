### If you want to add a feature/fix something:
* Make the feature/fix.
* Make sure it does not break anything.
* Make a PR.

### When making a PR:
* Make sure it does not break anything, or if it does, mention it in the description.

### Code formatting:
* Try to follow the style of pre-existing code:
  * 2 spaces for indentation
  * variable names and id names are `camelCase`
  * a single space between operators

### Use of AI:
* No AI slop.
* Make sure you understand and test AI code.
* Try to keep AI use minimal.
* Please mention its use in commit description.

### What is the dev branch for?
* When I make a change, I do it on the dev branch and then PR it into main.
* It doesn't have any other purpose.
